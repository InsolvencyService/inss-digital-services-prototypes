Common issues when uploading RP14/A forms

Even if you have followed all the steps in the upload instructions, you might see an error message. Using IP software to upload the data will minimise the risk of errors.

Common issues when uploading RP14/A forms:

File size

The file size must be less than 10MB. If your file is bigger than this, you need to separate it into smaller files so it can be uploaded.

Case and company details

These can cause delays in processing claims. To avoid this, check that:

  • the case reference number (CN) is present and correct, and that there are no spaces before or after it. The employer name and case reference must be entered on each line of the spreadsheet
  • the case reference number (CN) is present and correct, and that there are no spaces before or after it. The employer name and case reference must be entered on each line of the spreadsheet
  • the correct Company Registration Number (CRN) is used, check the CRN match es the case
  • National Insurance numbers are all valid and correct, as temporary or dummy numbers will not be accepted by the platform

Formatting problems

These can include:

  • Empty or blank rows at the end of the file. To remove any empty records:
    1. Highlight the blank row.
    2. Right click.
    3. Delete the row.
  • Too many decimal places. You might need to select each cell individually to make sure figures do not have more than 2 decimal places, as it’s not always visible.
    • In the RP14, you will need to check the Shareholder percentage field
    • In the RP14A, you will need to check 1). basic pay per week, 2). arrears of pay owed and 3). number of days holiday owed
    • Incorrect characters. Only numbers must be put in the numerical fields. If you include any special characters, like £, or any text, the upload will fail.
  • Incorrect date entries. Date fields must be formatted as DD/MM/YYYY. If you do not know the date and the field is not mandatory, leave it blank.
  • Overtyped drop-down data. When entering data into a cell with a drop-down box, you must select one of the options provided using the selector, and don’t overtype.
  • Ignoring the smart tag. Do not enter any data after the blue smart tag on the spreadsheet, as it will be lost. [
